  • Logic (e.g., propositional logic or first-order logic) is a language used to represent knowledge and to reason (draw conclusions, make inferences) based on this knowledge

  • Logic is used in a knowledge-based AI agent to represent knowledge and to reason (draw conclusions, make inferences) based on this knowledge

  • In deductive logic the goal is to determine whether an argument is valid (the premises entail the conclusion), using some inference algorithm or rules

  • In inductive logic the goal is to determine whether an argument is strong (the premises partially entail the conclusion), using some inference algorithm or rules

Propositional symbols/variables

In propositional logic, the terms propositional symbol and propositional variable are often used interchangeably to denote atomic propositions—statements that are either true or false and cannot be broken down into simpler statements. These atomic propositions serve as the foundational building blocks for more complex, compound propositions formed using logical connectives/boolean operators such as AND, OR, and NOT.

Logical probability function

prob <- function(..., H, p = NULL) {
  
  # Capture the premises as a list of quosures
  premises <- enquos(...)
  
  # Capture the conclusion as a quosure
  conclusion <- enquo(H)
  
  # Extract variable names from premises and conclusion
  vars <- unique(c(
    unlist(lapply(premises, all.vars)),
    all.vars(conclusion)
  ))
  
  # Generate truth table with all possible combinations of variables
  truth_table <- expand.grid(replicate(length(vars), c(TRUE, FALSE), simplify = FALSE))
  colnames(truth_table) <- vars
  
  # Evaluate each premise within the context of the truth table
  for (i in seq_along(premises)) {
    premise <- premises[[i]]
    truth_table[[paste0("P", i)]] <- eval_tidy(premise, data = truth_table)
  }
  
  # Determine if all premises are TRUE for each row
  truth_table <- truth_table %>%
    mutate(E = if_all(starts_with("P"), ~ .x))
  
  # Evaluate the conclusion within the context of each row
  truth_table <- truth_table %>%
    mutate(H = eval_tidy(conclusion, data = truth_table))
  
  # Assign uniform prior probabilities if none provided
  if (is.null(p)) {
    truth_table <- truth_table %>%
      mutate(p = 1 / n())
  } else {
    # Ensure that the provided probability vector matches the truth table rows
    if (length(p) != nrow(truth_table)) {
      stop("Length of probability vector 'p' must match the number of truth table rows.")
    }
    truth_table <- truth_table %>%
      mutate(p = p)
  }
  
  # Print truth table
  print(truth_table)
  
  # Calculate probability P = P(H | E)
  P <- truth_table %>%
    filter(E) %>%  # Select rows where all premises are TRUE
    summarise(P = sum(p * H) / sum(p)) %>% # Weighted average of H
    pull(P)
  
  return(P)
}

Modus ponens

Modus ponens is a deductive argument that affirms the antecedent to conclude the consequent

Argument Structure:

  • Premises:

    • P1: If it rains, the ground is wet (A => B)

    • P2: It is raining (A)

  • Conclusion:

    • H: The ground is wet (B)
P_modus_ponens <- prob(
  P1 = A %=>% B,    # P1: If A, then B
  P2 = A,           # P2: A is true
  H = B             # H: B is true
)

3  TRUE FALSE FALSE  TRUE FALSE FALSE 0.25

paste("Probability (P) of Modus Ponens:", P_modus_ponens)

Modus tollens

Modus tollens is a deductive argument that denies the consequent to conclude the negation of the antecedent

Argument Structure:

  • Premises:

    • P1: If it rains, the ground is wet (A => B)

    • P2: The ground is not wet (¬B)

  • Conclusion:

    • H: It did not rain (¬A)
P_modus_tollens <- prob(
  P1 = A %=>% B,    # P1: If A, then B
  P2 = !B,          # P2: Not B
  H = !A            # H: Not A
)
      A     B    P1    P2     E     H    p
1  TRUE  TRUE  TRUE FALSE FALSE FALSE 0.25
2 FALSE  TRUE  TRUE FALSE FALSE  TRUE 0.25
3  TRUE FALSE FALSE  TRUE FALSE FALSE 0.25
4 FALSE FALSE  TRUE  TRUE  TRUE  TRUE 0.25
paste("Probability (P) of Modus Tollens:", P_modus_tollens)
[1] "Probability (P) of Modus Tollens: 1"

Affirming the consequent

Affirming the consequent is an inductive argument that affirms the consequent to conclude the antecedent

Argument Structure:

  • Premises:

    • P1: If it rains, the ground is wet (A => B)

    • P2: The ground is wet (B)

  • Conclusion:

    • H: It rains (A)
P_affirming_consequent <- prob(
  P1 = A %=>% B,    # P1: If A, then B
  P2 = B,          # P2: Not B
  H = A            # H: Not A
)

3  TRUE FALSE FALSE FALSE FALSE  TRUE 0.25

Denying the antecedent

Denying the antecedent is an inductive argument that denies the antecedent to conclude the consequent

Argument Structure:

  • Premises:

    • P1: If it rains, the ground is wet (A => B)

    • P2: It does not rain (!A)

  • Conclusion:

    • H: The ground is wet (B)
P_denying_antecedent <- prob(
  P1 = A %=>% B,    # P1: If A, then B
  P2 = !A,           # P2: A is true
  H = B             # H: B is true
)
